# 📁 Структура файлов - Kitten Tennis

Полное описание каждого файла и что в нём находится.

## 🗂️ Корневые файлы

### `package.json`
**Что:** Конфигурация Node.js проекта  
**Для кого:** npm (менеджер пакетов)  
**Содержит:**
- Название и версию проекта
- Зависимости (ws - WebSocket)
- Скрипты запуска (`npm start`, `npm run dev`)

### `.gitignore`
**Что:** Список файлов, которые Git игнорирует  
**Для кого:** Git (система версионирования)  
**Содержит:**
- `node_modules/` (не загружаем в гит)
- `.env` файлы (приватные)
- Логи и кэш

### `README.md`
**Что:** Полное описание проекта  
**Для кого:** Людей на GitHub  
**Содержит:**
- Описание проекта
- Инструкция по запуску
- Инструкция по развертыванию на Render
- Управление и советы

### `SETUP.md`
**Что:** Быстрая инструкция по запуску  
**Для кого:** Тебя, для быстрого старта  
**Содержит:**
- Как запустить локально
- Как загрузить на GitHub
- Как развернуть на Render

### `CHANGELOG.md`
**Что:** История всех изменений  
**Для кого:** Разработчиков и пользователей  
**Содержит:**
- Что исправлено в версии 2.1
- Что было в версии 2.0
- Планы на будущее

### `FILES.md` (этот файл)
**Что:** Описание структуры проекта  
**Для кого:** Тебя, чтобы понять что где  
**Содержит:**
- Описание каждого файла
- Что в каком файле находится

## 🖥️ Запуск

### `start.bat`
**Что:** Скрипт быстрого запуска для Windows  
**Как использовать:** Просто двойной клик  
**Делает:**
- Проверяет наличие Node.js
- Устанавливает зависимости если нужно
- Запускает сервер
- Показывает адрес localhost:3000

### `start.sh`
**Что:** Скрипт быстрого запуска для Mac/Linux  
**Как использовать:** `bash start.sh` или `./start.sh`  
**Делает:** То же, что и start.bat

## 📂 Папка `server/`

### `server.js` (ГЛАВНЫЙ ФАЙЛ СЕРВЕРА)
**Что:** Node.js сервер с WebSocket  
**Размер:** ~8KB  
**Основные части:**

```javascript
// 1. HTTP сервер (раздача файлов)
http.createServer((req, res) => { ... })

// 2. WebSocket сервер
const wss = new WebSocket.Server({ server })

// 3. Класс Room (одна игровая комната)
class Room {
  tick(dt) { ... }      // обновление физики
  goal(scorer) { ... }  // обработка забитого гола
  broadcast(msg) { ... }  // отправка всем игрокам
}

// 4. WebSocket обработчик
wss.on('connection', ws => {
  ws.on('message', raw => { ... })
})
```

**Ключевые функции:**
- `startGame()` - начать игру
- `tick(dt)` - обновление физики (60 fps)
- `goal()` - обработка гола
- Коллизии мяча и игроков
- Bot AI (если режим с ботом)

## 📂 Папка `docs/` (ФРОНТЕНД)

Это папка со всеми файлами, которые видит браузер!

### `index.html` (ГЛАВНЫЙ HTML ФАЙЛ)
**Что:** Структура всей страницы  
**Размер:** ~2.5KB  
**Содержит:**

```
<div id="s-menu">         ← Экран меню
  <!-- Кнопки и поля -->

<div id="s-game">         ← Экран игры
  <canvas id="cv">        ← Основной канвас для отрисовки
  
<div id="s-over">         ← Экран конца игры
  <!-- Результаты -->

<div id="s-disc">         ← Экран отключения соперника
```

**Элементы:**
- `#inp-nick` - поле ввода ника
- `#btn-create` - кнопка создать комнату
- `#btn-bot` - кнопка играть с ботом
- `#btn-join-show` - кнопка присоединиться
- `#btn-skin1`, `#btn-skin2` - кнопки выбора скинов
- `#cv` - основной canvas для игры

### `game.js` (ГЛАВНАЯ ЛОГИКА ИГРЫ)
**Что:** Вся логика клиента (браузер)  
**Размер:** ~12KB  
**Основные классы:**

```javascript
// Физические объекты
class Ball { ... }        // мяч
class Player { ... }      // персонаж (игрок)
class Particles { ... }   // частицы (эффекты)
class Confetti { ... }    // конфетти при победе

// Системы
class Background { ... }  // фон и корт
class Game { ... }        // главная игровая логика

// UI
const UI = { ... }        // интерфейс меню

// Константы
const SKINS = { ... }     // скины персонажей
```

**Ключевые переменные:**
- `CURRENT_SKINS` - выбранные скины [P1, P2]
- `IS_BOT_MODE` - режим ли с ботом
- `NICKS` - ники игроков

**Основные функции:**
- `Game.tick()` - обновление игры каждый кадр
- `Game.draw()` - отрисовка всего на canvas
- `Game.start()` - начало игры
- `UI.init()` - инициализация интерфейса
- `SFX.play()` - воспроизведение звука

### `socket.js`
**Что:** WebSocket клиент для связи с сервером  
**Размер:** ~1KB  
**Содержит:**

```javascript
const SocketManager = {
  connect()       // подключиться к серверу
  send(obj)       // отправить сообщение
  on(type, fn)    // слушать событие с сервера
  create(nick)    // создать комнату
  join(code, nick) // присоединиться к комнате
  move(d)         // отправить позицию
  rematch()       // реванш
}
```

**События с сервера:**
- `created` - комната создана
- `joined` - присоединился к комнате
- `start` - игра началась
- `ball` - обновление позиции мяча
- `hit` - кто-то ударил мяч
- `score` - кто-то забил
- `gameover` - игра закончилась
- `left` - соперник вышел

## 📊 Взаимосвязь файлов

```
Браузер открывает index.html
    ↓
index.html подключает:
    - style.css (дизайн)
    - socket.js (подключение к серверу)
    - game.js (логика игры)
    ↓
game.js подключает:
    - SocketManager из socket.js
    - Определяет классы Game, Player, Ball
    ↓
socket.js подключает:
    - Сервер на http://localhost:3000 (или https://...)
    ↓
server.js слушает:
    - HTTP запросы (раздача файлов из docs/)
    - WebSocket соединения
    ↓
Игроки отправляют друг другу данные через WebSocket
```

## 🔄 Поток данных

### При создании комнаты:
```
Client (socket.js)  →  Server (server.js)
  create({ nick })  →  Room создаётся
                    ←  created({ code, pi: 0 })
```

### Во время игры:
```
Server (game.tick)  →  Client (game.onBall)
  ball({ bx, by })  →  Ball обновляет позицию
  hit({ ... })      →  Воспроизведение звука

Client (Game.update) → Server (socket.move)
  move({ x, y, ... }) → Сервер обновляет позицию
                     ← pmove({ pi, x, y })
                        Второй игрок получает данные
```
